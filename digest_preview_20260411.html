PSI: Shared State as the Missing Layer for Coherent AI-Generated Instruments in Personal AI Agents

Zhiyuan Wang, Erzhen Hu, Mark Rucker · arXiv · 2026-04-09

WHAT IT IS PSI is a shared-state architecture for AI-generated personal software. Instead of LLM-generated tools existing as isolated apps, PSI gives them a common "context bus" — a layer where each module publishes its current state and exposes write-back hooks, so they can talk to each other and to a central chat agent.

WHY IT'S INTERESTING Right now, every AI-generated tool is an island. PSI proposes the plumbing contract that lets them become a coherent environment — new modules auto-integrate just by following the same spec. The researcher actually ran this on themselves for three weeks, which is a better stress test than most papers bother with.

WORTH WATCHING BECAUSE As on-demand UI generation gets good, the fragmentation problem becomes real fast. Whoever defines the interoperability contract early wins. This is a concrete stab at that layer before the mess sets in.
